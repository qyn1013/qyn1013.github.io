Package.json

2019-08-05 阅读次数：
目录
回到顶部

package.json定义项目所需的模块和项目的配置信息
用npm init初始化一个package.json文件

  1. name

    1.会作为URL的一部分、命令行的参数或文件夹的名字
    2.不能以.或_开头，不能有大写字母

  2. version

    必须由node-semver解析，与npm一起作为依赖项

  3. main

    主要字段是模块 ID，是程序的主要入口点,模块被引入后，默认首先加载index.js

  4. ~/^版本号的区别

    //匹配最近的小版本依赖包，例如5.0.3是安装5.0.x中最新的版本
    ^//匹配最新的大版本依赖包，例如5.0.3是安装5.x.x中最新的版本
    用    来标记版本号

  5. dependencise和devDependencise的区别

    devDependencise只用于本地环境开发
    dependencise需要发布到生产环境

  6. semver-语义化版本控制规范

    版本号采用X.Y.Z的格式，X,Y,Z为非负整数，每个元素以数值递增
    npm install –save semver
    patch//内部修复
    major//主要更新不兼容，无法在不更改软件的情况下进行升级
    minor//次要更新向后兼容

  7. scripts

    指定运行脚本命令的npm命令行缩写
