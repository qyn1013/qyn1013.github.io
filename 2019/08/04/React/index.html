1.JSX

在属性中嵌入JavaScript表达式时，仅使用引号（对于字符串）或大括号（对于表达式），对于同一属性不能同时使用引号和大括号

使用camelCase定义属性的名称

2.元素渲染

setInterval（）方法重复调用一个函数或执行一个代码段，返回一个intervalID

React只更新它需要更新的部分

通过调用ReactDOM.render()来修改想要渲染的元素

3.组件&Props

接受任意的入参（“props”），返回用于描述页面展示内容发React元素

组件名称开头大写，以小写字母开头的组件视为DOM标签

Props 只读性,组件使用函数声明和通过class声明都不能修改自身的props

4.State&生命周期

https://www.runoob.com/react/react-component-life-cycle.html

componentWillMount():组件即将被渲染到页面之前出发，可以进行开启定时器、向服务器发送请求等操作

render():组件渲染

componentDidMount():组件已经被渲染到页面中后触发：可以进行DOM相关的操作

运行中阶段
componentWillReceiveProps()：组件接收到属性时触发
shouldComponentUpdate()：组件收到新属性或组件状态发生改变时触发（首次渲染不会被触发）

state是私有的，完全受控于当前组件

componentDidMount():”挂载（mount）”
componentWillUnmount():”卸载（umount）”

setState()：时刻更新组件state
1.不要直接修改State，构造函数是唯一可以给this.state赋值的地方

2.可能是异步更新,让 setState() 接收一个函数而不是一个对象。这个函数用上一个 state 作为第一个参数，将此次更新被应用时的 props 做为第二个参数

3.State更新会被合并，当调用setState()时，React会把所提供的对象合并到当前state

6.Key

一个元素的key最好是这个元素拥有的一个独一无二的字符串。
如果用index作为key值，会导致性能变差

如果提取出一个listitem组件，应该把key放在数组中的元素上，而不是ListItem组件中的<li>元素上

判断数据是否属于state

是否由父组件通过props传递而来的
是否随事件的推移保持不变
能否根据其他state或props计算出该数据的值

任何可变数据应该只有一个对应的唯一的“数据源”

class Example extends React.Component {
  constructor(props) {
    super(props);
    this.state = {
      count: 0
    };
  }

  render() {
    return (
      <div>
        <p>You clicked {this.state.count} times</p>
        <button onClick={() => this.setState({ count: this.state.count + 1 })}>
          Click me
        </button>
      </div>
    );
  }
}
